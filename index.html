01. Introducción

¿Qué es Linux?

Linux no es solo un sistema operativo, es una filosofía de software libre y código abierto que ha revolucionado la tecnología.

🐧

Definición

Linux es un sistema operativo tipo Unix de código abierto que se distribuye bajo la licencia GPL (General Public License). Fue creado por Linus Torvalds en 1991 como un proyecto personal mientras estudiaba en la Universidad de Helsinki.

A diferencia de Windows o macOS, Linux es modular: su corazón es el Kernel (núcleo), al cual se le añaden herramientas y aplicaciones para formar una distribución completa.

Línea de Tiempo

1969

Nace UNIX

AT&T Bell Labs desarrolla UNIX, el precursor de todos los sistemas tipo Unix.

1983

Proyecto GNU

Richard Stallman inicia el proyecto GNU con el objetivo de crear un sistema operativo libre.

1991

Nace Linux

Linus Torvalds publica la primera versión del kernel Linux (0.01).

1993

Primeras Distros

Aparecen Slackware y Debian, las primeras distribuciones completas.

2004

Ubuntu

Canonical lanza Ubuntu, revolucionando la accesibilidad de Linux.

2026

Presente

Linux domina servidores, supercomputadoras, Android y la nube.

🔓

Libertad

Usa, estudia, modifica y distribuye el software libremente.

👁️

Transparencia

Código fuente disponible para auditación y mejora colectiva.

🤝

Comunidad

Desarrollo colaborativo por miles de programadores worldwide.

🔧

Personalización

Adapta cada aspecto del sistema a tus necesidades específicas.

= GNU/Linux

El sistema operativo completo es la combinación del Kernel Linux + las herramientas del proyecto GNU + software adicional. Por eso muchos prefieren llamarlo GNU/Linux en lugar de solo Linux, reconociendo el trabajo de la Free Software Foundation.

¿Por qué "GNU/Linux" y no solo "Linux"?

01

Richard Stallman y la FSF argumentan que llamarlo solo "Linux" ignora el trabajo de GNU que hace el sistema usable.

02

El kernel Linux representa aproximadamente el 2% del código total de un sistema operativo típico.

03

En la práctica, la mayoría dice "Linux" por simplicidad, pero es importante conocer la historia completa.

08. Aplicaciones

Casos de Uso

Linux está en todas partes. Descubre dónde brilla este sistema operativo.

🖥️

Servidores

El 96.3% de los servidores top 1 millón ejecutan Linux. Su estabilidad, seguridad y eficiencia lo hacen ideal para infraestructura crítica.

  • Uptime de años sin reiniciar
  • Gestión remota eficiente
  • Virtualización nativa
  • Contenedores (Docker, Kubernetes)
💻

Desarrollo

El entorno preferido por desarrolladores. Herramientas nativas, compatibilidad total con estándares y sin restricciones.

  • Git nativo
  • Containers sin overhead
  • Lenguajes preinstalados
  • IDEs potentes (VS Code, JetBrains)
🎮

Gaming

Proton y Steam han revolucionado el gaming en Linux. 80%+ de juegos Steam son jugables, muchos con rendimiento nativo.

  • Steam + Proton
  • Lutris para launchers
  • Mejor rendimiento en algunos títulos
  • Anti-cheat funcionando
🔒

Ciberseguridad

Desde Kali Linux para pentesting hasta Qubes OS para anonimato. Control total sobre tu privacidad.

  • Kali Linux (pentesting)
  • Tails (anonimato)
  • Qubes OS (aislamiento)
  • WireGuard/OpenVPN nativos
🤖

IoT y Embedded

Desde Raspberry Pi hasta supercomputadoras. Linux domina el Internet de las Cosas y sistemas embebidos.

  • Raspberry Pi OS
  • Arduino IDE
  • ROS (robots)
  • Contenedores ligeros (Alpine)
🎨

Creatividad

Suite completa de software creativo open source. Blender, GIMP, Inkscape, Ardour y más a nivel profesional.

  • Blender (3D, industria)
  • GIMP/Photoshop alternativas
  • DaVinci Resolve (edición video)
  • Ardour/LMMS (audio)
📱

Android

Sí, Android usa el kernel Linux. Es la prueba del dominio de Linux en dispositivos móviles.

  • 2.5B+ dispositivos activos
  • 70%+ del mercado móvil
  • Kernel Linux modificado
  • Apps nativas en Java/Kotlin
☁️

Cloud & Big Data

AWS, Google Cloud, Azure... todos corren Linux. Hadoop, Spark y toda la infraestructura de datos moderna.

  • AWS EC2 (Linux por defecto)
  • Kubernetes (creado por Google)
  • Hadoop/Spark
  • Serverless functions
09. Terminal

Comandos Esenciales

La terminal es tu superpoder. Domina estos comandos y controla Linux como un profesional.

user@linux-hub:~

$ # Bienvenido a GNU/Linux Hub - Guía de Comandos

📁 Navegación

ls Listar archivos y directorios
cd <directorio> Cambiar de directorio
pwd Mostrar directorio actual
tree Mostrar árbol de directorios

📄 Archivos

cp <origen> <destino> Copiar archivos
mv <origen> <destino> Mover o renombrar
rm <archivo> Eliminar archivo
mkdir <nombre> Crear directorio
touch <archivo> Crear archivo vacío

🔐 Permisos

chmod <permisos> <archivo> Cambiar permisos
chown <usuario> <archivo> Cambiar propietario
sudo <comando> Ejecutar como root

⚙️ Sistema

ps aux Listar procesos
top Monitor de recursos
df -h Espacio en disco
du -sh <directorio> Tamaño de carpeta
free -h Memoria RAM

🌐 Red

ping <host> Test de conectividad
ip addr Configuración de red
curl <url> Descargar/Transferir
wget <url> Descargar archivos
ss -tuln Puertos abiertos

📦 Paquetes

apt install <paquete> Instalar (Debian/Ubuntu)
dnf install <paquete> Instalar (Fedora/RHEL)
pacman -S <paquete> Instalar (Arch)
snap install <app> Instalar Snap
flatpak install <app> Instalar Flatpak

Depende de tus objetivos: Para usarlo como escritorio cotidiano, unos días. Para sentirte cómodo con la terminal, semanas. Para dominar la administración de sistemas, meses o años. La curva de aprendizaje inicial es mayor que Windows, pero las recompensas en control y eficiencia son inmensas.
